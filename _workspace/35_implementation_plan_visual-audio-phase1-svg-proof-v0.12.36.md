# Implementation Plan — Visual/audio Phase 1.2 SVG proof v0.12.36

## Task restatement

Prove that the selected flat institutional vocabulary can render a deterministic,
keyboard-reachable, text-equivalent SVG scene from a visible fixture without
creating a second simulation or legality engine.

## Minimal implementation plan

1. Define a narrow visible scene model with identity/marker/status fallbacks.
2. Render stable SVG cards and facility controls with labels, symbols, and
   uncertainty patterns.
3. Add a static proof page with local institution/facility selection and
   reduced-motion control.
4. Add deterministic snapshot, XML/static boundary, keyboard, fallback, and
   render-time tests; register the renderer and align SDD/roadmap docs.

## Acceptance criteria

- All Phase 1.2 checklist items have committed technical evidence.
- Scene output is stable for the fixture and contains no hidden-state input.
- Existing live GUI and host/simulation behavior are unchanged.

## Non-goals and stop conditions

No live DTO, browser state model, command/transition, true geography, audio,
animation, or human accessibility claim. Stop if proof integration requires
new host fields or a second authority boundary.
