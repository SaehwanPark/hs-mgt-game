# Request Summary — Visual/audio Phase 5.1 semantic containers v0.12.66

## Authorized outcome

Complete the Phase 5.1 semantic information-container checklist while
preserving the existing actor-visible GUI and host authority.

## Slice boundary

- Define eight restrained visual information classes: board packet, operations
  ledger, intelligence report, regulatory letter, project sheet, news wire,
  executive action queue, and after-action report.
- Document structural header treatment, non-color marker, semantic purpose,
  compact/expanded variants, accessibility, large-text, narrow-width, print,
  reduced-motion, and source/status behavior in one deterministic catalog.
- Apply those classes to existing GUI panels and provide a fixture-only proof
  page without adding host calls, client-side state, or hidden information.
- Preserve the shared grid, current text density, source/status language,
  actor-visible boundaries, keyboard access, and exact visible values.
- Do not change Rust DTOs, simulation state, commands, transitions, stochastic
  inputs, history, hashes, replay authority, audio, or debrief behavior.

Focused semantic-container, GUI, registry, credits, metadata,
documentation-link, full Python, full Rust, formatting, presentation-contract,
and diff checks.

authority boundaries only. They do not establish human usability, lived
accessibility, contrast, learning, calibration, browser behavior, or policy
validity.
